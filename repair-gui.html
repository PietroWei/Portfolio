Internal Process Management GUI

A Python and PyQt5 desktop tool that replaced a legacy Excel workflow for a team of helicopter repair engineers.

Overview

The application replaced the legacy Excel workflow used by more than ten engineers to manage repair cases for international helicopter operators. The previous process meant juggling spreadsheets, losing context between cases, and routing simple technical questions through long email chains. The new tool consolidates intake, diagnosis, repair and closure into one Python desktop application backed by a shared SQLite database, with 3D STL visualization of components and automatic generation of PDF and Word reports.

Note: the source repository is private because the tool is used internally and embeds proprietary logic. This page collects the highlights so the work is still readable from the outside.

Problem and Solution

Problem

  • Repair cases tracked across multiple Excel files and shared drives, with no single source of truth
  • Duplicate entries and lost context whenever a case was handed over between team members
  • Technical questions routed through long email chains instead of a structured record
  • Reporting required extracting and reshaping data manually in Excel every week
  • Part of the workflow depended on an external software that was both expensive and underperforming

Solution

  • One desktop application with a tabbed workflow covering intake, diagnosis, repair and closure
  • Shared SQLite backend with concurrent multi-user access and a normalized schema
  • Multi-parameter search across open and historical cases in under a second
  • Interactive 3D STL viewer so engineers can reference component geometry inline with the case
  • One-click generation of PDF and Word repair reports from case data
  • In-app analytics dashboards plus Power BI ready exports for fleet KPI and SLA monitoring

Tech Stack

  • Language: Python 3.11
  • GUI: PyQt5
  • Database: SQLite with a shared file layout supporting concurrent readers and serialized writers
  • 3D viewer: STL rendering via the pyvista / VTK family
  • Reporting: openpyxl, python-docx, reportlab
  • Analysis: pandas, NumPy
  • Packaging: PyInstaller single-file Windows executable
  • Testing: pytest on the data access and reporting layers

Usage Example

  1. Open a case. The engineer searches by serial number, part number or case id. The app queries SQLite and matching cases appear instantly with last known configuration and prior operator.
  2. Log the diagnosis. Free-text notes plus a controlled list of fault codes keep statistics clean. Reference geometry is pulled directly from the 3D STL viewer on the same screen.
  3. Propose the repair. Parts are selected from a searchable catalog and totals update live. Approvals are logged with user, timestamp and reason.
  4. Close the case. The record is frozen, a PDF repair report is generated, a Word client note is produced from a template, and the weekly KPI view is refreshed automatically.

Impact

  • About €100k of outsourcing cost avoided by moving off the external software
  • Roughly 30% reduction in processing time per repair case
  • Fewer data-entry errors thanks to controlled vocabularies and foreign-key constraints at the database level
  • New reports are now a matter of writing a SQL view instead of filing a vendor change request
  • Single tool adopted by the full team of engineers, replacing the previous mix of Excel files and external software
